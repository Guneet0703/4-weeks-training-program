# Day 20 - Introduction to Node.js

## Overview

The twentieth day of the MERN Stack training marked the beginning of back-end development with Node.js. The session introduced Node.js, its architecture, advantages, installation process, and how JavaScript can be executed outside the browser. The session also covered npm (Node Package Manager), Node.js modules, and the execution of JavaScript files using Node.js. These concepts form the foundation for developing scalable server-side applications in the MERN Stack.

---

## Topics Covered

### 1. Introduction to Node.js

Studied Node.js, an open-source JavaScript runtime environment built on Chrome's V8 JavaScript Engine. Understood how Node.js allows JavaScript to run outside the browser and enables developers to build fast and scalable server-side applications.

---

### 2. Features of Node.js

Learned the major features of Node.js, including:

- Asynchronous and Event-Driven Architecture
- Non-Blocking I/O Operations
- Fast Execution using the V8 JavaScript Engine
- Single-Threaded Event Loop
- Cross-Platform Support
- Highly Scalable
- Lightweight Runtime Environment

These features make Node.js suitable for modern web application development.

---

### 3. Installing Node.js

Learned how to install Node.js and verify the installation using terminal commands.

**Commands Practiced:**

```bash
node -v
npm -v
```

These commands display the installed versions of Node.js and npm.

---

### 4. Node Package Manager (npm)

Studied npm (Node Package Manager), the default package manager used to install, update, and manage project dependencies.

**Common Commands:**

```bash
npm init
npm install
npm uninstall
```

Also understood the purpose of the `package.json` file in a Node.js project.

---

### 5. Running JavaScript using Node.js

Learned how to execute JavaScript files using the Node.js runtime.

**Command:**

```bash
node filename.js
```

This command executes a JavaScript file outside the browser using Node.js.

---

### 6. Node.js Modules

Studied the concept of modules in Node.js and understood how code can be organized into separate files for better maintainability and reusability.

**Keywords Covered:**

- `require()`
- `module.exports`

Modules help organize applications into reusable components.

---
